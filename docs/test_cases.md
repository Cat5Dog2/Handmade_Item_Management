# ハンドメイド在庫・販売管理アプリ テストケース一覧

## 1. 目的

本書は、MVP向けに定義されたテスト観点を、実施可能なテストケースへ具体化したものである。  
対象範囲は、認証、商品管理、商品画像、タスク、カテゴリ・タグ、QR販売済更新、ダッシュボード、共通エラー表示、主要API、主要ログ、および一部非機能確認とする。

## 2. 前提

- 対象リリース: MVP 初期リリース
- 日時表示は JST とする
- 商品は一点物であり、数量在庫は扱わない
- 商品削除は論理削除、タスク・カテゴリ・タグ削除は物理削除とする
- 商品画像URLは期限付きURLであり、期限切れ後は再取得で更新する

## 3. テストデータ前提

- 認証用アカウントが存在すること
- 以下の状態の商品を用意すること
  - 制作前 / 制作中 / 制作済 / 展示中 / 在庫中 / 販売済
  - 論理削除済み商品
- 以下の画像状態の商品を用意すること
  - 画像0枚
  - 画像1枚
  - 画像複数枚
  - 代表画像あり / なし
  - 画像10枚登録済み
- 以下のタスク状態を用意すること
  - 未完了
  - 完了済み
  - 納期あり
  - 納期なし
- 使用中 / 未使用のカテゴリ・タグを用意すること
- 有効 / 無効 / 未登録 / 論理削除済み / 販売済み商品のQRコードを用意すること
