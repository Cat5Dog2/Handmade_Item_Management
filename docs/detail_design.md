# **2. 共通詳細設計**

## **2.1 アーキテクチャ詳細**

フロントエンドは Firebase Authentication SDK
により認証状態を監視し、保護対象画面表示時に ID トークンを取得して API
呼び出しへ付与する。バックエンドは Express
ミドルウェアでトークンを検証し、認証済み要求のみ業務処理へ渡す。商品・カテゴリ・タグ・タスクは
Firestore、画像は Cloud Storage へ保存する。

|-----------------|----------------------------------------------------------------------|--------------------|

## **2.3 共通入力・正規化**

| **項目種別**   | **正規化/制約**                                                                | **備考**                          |
|----------------|--------------------------------------------------------------------------------|-----------------------------------|
| 名称系         | 前後空白除去、空白のみ禁止、改行禁止、タブ禁止、制御文字禁止                   | 商品名/タスク名/カテゴリ名/タグ名 |
| 複数行テキスト | 最大文字数判定、改行は LF に統一、制御文字は除去または拒否                     | 商品説明/タスク内容/メモ          |
| 検索キーワード | 前後空白除去、連続空白を単一空白相当、英字小文字化、全半角差を可能な範囲で吸収 | 部分一致検索                      |
| 価格           | 0 以上整数、小数不可、数値以外不可                                             | 円単位                            |
| 画像           | JPEG/PNG/WebP、10MB 以下、最大 10 枚、長辺 2000px 超は縮小保存                 | 元画像は保持しない                |

## **2.4 認証・認可**

| **項目**       | **内容**                                                  |
|----------------|-----------------------------------------------------------|
| 認証方式       | Firebase Authentication のメールアドレス + パスワード認証 |
| API 認証       | 業務APIは Authorization: Bearer {Firebase ID Token} を必須とする。`GET /api/health` は運用確認用のため認証不要とする |
| 認可           | 本人のみ利用前提のためロール判定は設けない                |
| 未認証アクセス | 保護対象画面へは遷移不可。ログイン画面へリダイレクトする  |
| ログアウト     | Firebase セッション破棄 + ローカル UI / 画面状態 / 検索条件キャッシュ破棄。URL上に過去の検索条件が残っていても、ログアウト直後の再ログインでは復元対象にしない |


補足:

- 商品一覧の検索条件を URL クエリで保持している場合でも、ログアウト直後の再ログインでは復元しない
- ログアウト後の初期遷移先はダッシュボードとし、商品一覧へ再訪した場合も初期条件から開始する

> **•** 通常 API では、論理削除済み商品およびそれに紐づく関連リソースへのアクセスは HTTP 404 を返す。
> **•** エラーコードは `PRODUCT_DELETED` / `PRODUCT_RELATED_RESOURCE_UNAVAILABLE` で業務上の内訳を区別してよいが、HTTP ステータスは 404 に統一する。
> **•** ただし `POST /api/qr/lookup` は読取結果照会APIのため例外とし、未登録商品・論理削除済み商品も HTTP 200 で `reasonCode` により返却する。

## **2.7 日時・業務日付**

| **項目** | **設計**                                                                              |
|----------|---------------------------------------------------------------------------------------|
| 保存     | Firestore には UTC の Timestamp として保存する。                                      |
| 表示     | 画面表示は JST (UTC+09:00) に変換して YYYY/MM/DD HH:mm または YYYY/MM/DD で表示する。 |
| 納期判定 | 納期が近いタスク判定は JST 基準の当日を含む 7 日以内とする。                          |
| 販売日時 | 販売済へ遷移した時点で未設定なら設定する。既に設定済みなら上書きしない。              |

# **3. 画面詳細設計**

本章では画面ごとの表示項目、初期表示、イベント、成功時/失敗時の挙動を定義する。実装では画面
ID をルーティング単位およびテストケース識別子として利用する。

## **3.1 SCR-01 ログイン画面**

| **目的**     | 利用者本人の認証を行い、認証成功時にダッシュボードへ遷移させる。 |
|--------------|------------------------------------------------------------------|
| **初期表示** | 認証中はボタン押下不可。通信失敗時は再操作可能状態へ戻す。       |

### **主な表示項目**

| **項目**         | **UI 種別**    | **詳細**                                     |
|------------------|----------------|----------------------------------------------|
| メールアドレス   | テキスト入力   | 必須。メール形式。                           |
| パスワード       | パスワード入力 | 必須。入力文字はマスク表示。                 |
| ログインボタン   | 主要ボタン     | 入力妥当時のみ活性。                         |
| パスワード再設定 | テキストリンク | ログイン画面内の再設定ダイアログを開く。 |
| エラーメッセージ | アラート       | 認証失敗、通信失敗時に表示。                 |

### **イベント/処理**

| **操作** | **契機/処理**                                | **結果**                                                           |
|----------|----------------------------------------------|--------------------------------------------------------------------|
| 初期表示 | 未ログインで本画面を表示                     | 前回入力値は初期化。セッション切れ遷移時のみ理由メッセージを表示。 |
| ログイン | Firebase signInWithEmailAndPassword 実行     | 成功時はダッシュボードへ。失敗時は認証失敗メッセージ表示。         |
| 再設定   | メールアドレス入力済ならその値を初期値に利用 | 送信成功メッセージまたは失敗理由を表示。                           |

## **3.2 SCR-02 ダッシュボード画面**

| **目的**     | ステータス別商品件数、販売済件数、未完了タスク件数、納期が近いタスク、最近更新商品を俯瞰表示する。 |
|--------------|----------------------------------------------------------------------------------------------------|
| **初期表示** | 0件でも件数カードは 0 を表示。リスト0件時は対象なしメッセージ。                                    |

### **主な表示項目**

| **項目**         | **UI 種別** | **詳細**                                           |
|------------------|-------------|----------------------------------------------------|
| ステータス別件数 | 件数カード  | 論理削除済み商品を除外して集計。                   |
| 販売済件数       | 件数カード  | status=sold の件数。                               |
| 未完了タスク件数 | 件数カード  | 論理削除されていない商品に紐づく未完了タスクのみ。 |
| 納期が近いタスク | リスト      | 当日含む7日以内の未完了タスク。                    |
| 最近更新した商品 | リスト      | 更新日時降順・最大5件。                            |

### **イベント/処理**

| **操作**     | **契機/処理**        | **結果**                                                           |
|--------------|----------------------|--------------------------------------------------------------------|
| 初期取得     | GET /api/dashboard   | 成功時に各カードとリストへ反映。                                   |
| 商品詳細遷移 | 最近更新商品の行押下 | 対象商品の商品詳細へ遷移。                                         |
| タスク管理遷移 | 納期が近いタスク押下 | 該当商品のタスク管理画面へ遷移。対象商品コンテキストを保持する。 |
| 再試行       | 取得失敗時           | 同一要求で再送。                                                   |

## **3.3 SCR-03 商品一覧画面**

| **目的**     | 商品の検索、絞り込み、並び替え、商品詳細遷移、新規登録遷移を行う。 |
|--------------|--------------------------------------------------------------------|
| **初期表示** | 再試行時は直前の条件とページを保持。0件時は条件クリア導線を表示。status=sold 選択時は販売済表示切替を ON 固定とする。 |

### **主な表示項目**

| **項目**       | **UI 種別**     | **詳細**                                                             |
|----------------|-----------------|----------------------------------------------------------------------|
| キーワード     | 検索入力        | 最大100文字。商品名/説明/productId/カテゴリ名/タグ名を対象。         |
| カテゴリ       | 単一選択        | 未選択可。                                                           |
| タグ           | 単一選択        | 未選択可。                                                           |
| ステータス     | 単一選択        | 未選択可。                                                           |
| 並び替え       | セレクト        | updatedAt / name、昇順降順。                                         |
| 販売済表示切替 | トグル          | 初期値 ON。OFF の場合 sold を除外。status=sold 選択時は ON 固定。    |
| 一覧           | カード/テーブル | thumbnail、商品名、productId、ステータス、カテゴリ、更新日時を表示。 |
| 新規登録       | 主要ボタン      | 商品登録画面へ遷移。                                                 |

### **イベント/処理**

| **操作**      | **契機/処理**           | **結果**                                                                  |
|---------------|-------------------------|---------------------------------------------------------------------------|
| 初期表示      | 条件なしで取得          | page=1, pageSize=50, sortBy=updatedAt, sortOrder=desc, includeSold=true。 |
| 検索/絞込変更 | 条件変更時に再取得      | page を 1 に戻す。条件は AND で判定。status=sold 選択時は includeSold を内部的に true として再取得する。 |
| 並び替え変更  | 選択変更                | page=1 に戻して再取得。                                                   |
| 詳細遷移      | 一覧行押下              | 商品詳細画面へ遷移。                                                      |
| 条件クリア    | 0件時またはユーザー操作 | キーワード・絞込・並び替えを初期値へ戻す。                                |

## **3.4 SCR-04 商品詳細画面**

| **目的**     | 商品の基本情報、画像、関連タスク、QR コードを表示し、QR 読み取り・編集・削除・タスク管理へ遷移する。 |
|--------------|-----------------------------------------------------------------------------------------|
| **初期表示** | 商品未存在または論理削除済みなら取得不可メッセージと一覧戻り導線を表示。                |

### **主な表示項目**

| **項目**   | **UI 種別**  | **詳細**                                                            |
|------------|--------------|---------------------------------------------------------------------|
| 基本情報   | 表示ラベル   | productId、名称、説明、価格、カテゴリ、タグ、ステータス、販売日時。 |
| 代表画像   | 画像         | 未設定時は sortOrder 最小画像、0件ならプレースホルダー。            |
| 画像一覧   | サムネイル列 | 最大10枚。                                                          |
| 関連タスク | 一覧         | 未完了優先表示。完了済み切替あり。各行で完了/未完了を直接切替可能。  |
| QRコード   | 画像         | qrCodeValue を元に生成したコードを表示。                            |
| QR読み取り | ボタン       | QRコード読み取り画面へ。                                            |
| 編集       | ボタン       | 商品登録/編集画面へ。                                               |
| 削除       | 危険ボタン   | 共通確認ダイアログを表示。                                          |
| タスク管理 | ボタン       | タスク管理画面へ。                                                  |

### **イベント/処理**

| **操作**       | **契機/処理**                                                                 | **結果**                                           |
|----------------|-------------------------------------------------------------------------------|----------------------------------------------------|
| 初期取得       | GET /api/products/:productId と GET /api/products/:productId/tasks?showCompleted=false を実行 | 成功時に基本情報、画像、QR、関連タスクを描画。     |
| QR読み取り     | QR読み取りボタン押下                                                          | QRコード読み取り画面へ遷移。                       |
| タスク完了切替 | 関連タスク行のチェック操作                                                    | PATCH /api/tasks/:taskId/completion 実行後、対象行の完了状態と completedAt のみ更新。 |
| タスク再取得   | 完了済み表示切替                                                              | GET /api/products/:productId/tasks を再実行し、関連タスク表示条件のみ切替。 |
| 削除           | 確認後 DELETE /api/products/:productId                                        | 成功時は一覧へ戻り完了メッセージを表示。           |

## **3.5 SCR-05 商品登録/編集画面**

| **目的**     | 商品の新規登録・更新および画像追加・代表画像設定・ステータス変更を行う。 |
|--------------|--------------------------------------------------------------------------|
| **初期表示** | 通信失敗時は入力値を保持。新規で未保存のまま画像即時アップロードは不可。 |

### **主な表示項目**

| **項目**   | **UI 種別**  | **詳細**                             |
|------------|--------------|--------------------------------------|
| 商品ID     | 表示のみ     | 新規時は未採番、保存後に表示。       |
| 商品名     | 単一行入力   | 必須、最大100文字。                  |
| 商品説明   | 複数行入力   | 任意、最大2000文字。                 |
| 価格       | 数値入力     | 必須、0以上整数。                    |
| カテゴリ   | 単一選択     | 必須。                               |
| タグ       | 複数選択     | 任意。                               |
| 商品画像   | アップロード | 最大10枚。JPEG/PNG/WebP。新規未保存時は非活性。 |
| 代表画像   | ラジオ/選択  | 登録済み画像から1件選択。新規未保存時は非表示または非活性。 |
| ステータス | 単一選択     | 必須。販売済から戻す場合は確認必須。 |
| 保存       | 主要ボタン   | 入力エラーが無い場合のみ活性。       |

### **イベント/処理**

| **操作**     | **契機/処理**                                   | **結果**                               |
|--------------|-------------------------------------------------|----------------------------------------|

# **4. API 詳細設計**

本章の業務APIは HTTPS/JSON
を前提とし、認証済みトークンを必須とする。`GET /api/health` は運用確認用のため認証不要とする。時刻はレスポンスでは ISO 8601
文字列、画面表示時に JST へ変換する。

## **4.1 API 共通仕様**

| **項目**         | **内容**                                                                                     |
|------------------|----------------------------------------------------------------------------------------------|
| 認証ヘッダ       | 業務APIは Authorization: Bearer {Firebase ID Token}。`GET /api/health` は不要                |
| Content-Type     | application/json（画像アップロード系は multipart/form-data を許容）                          |
| 文字コード       | UTF-8                                                                                        |
| 正常時ステータス | GET/PUT/DELETE は 200、POST 新規登録は 201 を基本とする                                      |
| 一覧メタ情報     | ページングを行う一覧 API のみ `page`, `pageSize`, `totalCount`, `hasNext` を返す             |
| 補足             | MVP では `GET /api/products` のみ対象。ページングしない一覧 API は `data.items` のみ返し、`meta` は省略可 |
| エラー本文       | code, message を必須。項目エラーがある場合のみ details を返す                                |

## **4.4 商品画像 API**

> **•** 画像追加・差し替え・削除 API は、期限付き URL の再生成を伴うため `displayUrl` / `thumbnailUrl` / `urlExpiresAt` をレスポンスに含めない。
> **•** フロントは各 API 成功後に `GET /api/products/:productId` を再取得し、最新の画像 URL・代表画像状態・sortOrder を再描画する。


### **4.4.1 POST /api/products/:productId/images**

| **認証** | 必須                            |
|----------|---------------------------------|
| **パス** | /api/products/:productId/images |
| **用途** | 商品画像の追加                  |

| **項目**  | **型** | **必須** | **説明**                 |
|-----------|--------|----------|--------------------------|
| file      | binary | 必須     | JPEG/PNG/WebP、10MB 以下 |

| **業務ルール** | **内容**                                                                 |
|----------------|--------------------------------------------------------------------------|
| 件数上限       | 既存含め最大10枚                                                         |
| 変換           | 長辺2000px超は縮小、display と thumbnail を生成                          |

# **5. データ詳細設計**

Firestore
は商品、タスク、カテゴリ、タグをコレクション単位で管理する。商品画像は
products ドキュメント内の埋め込み配列 images
とし、表示順と代表画像情報を保持する。

| **コレクション** | **用途**       | **主キー/識別子** |

| categories       | カテゴリマスタ | categoryId        |

| counters         | 連番管理       | counterKey        |
| operationLogs    | 主要操作ログ   | logId             |

# **6. 主要処理シーケンス**

## **6.1 商品新規登録**

> **•** フロントで必須・文字数・価格形式をチェックする。
> **•** POST /api/products を送信する。
> **•** API でカテゴリ/タグ存在確認と再バリデーションを行う。
> **•** counters/product をトランザクション更新して productId を採番する。
> **•** products ドキュメントを保存する。
> **•** レスポンス受領後は採番済み productId を用いて必ず商品詳細画面へ遷移する。以後の画像追加は保存済み商品として商品詳細画面または編集画面から実行する。新規保存前の画像アップロードは行わない。

## **6.2 商品更新**

> **•** 編集前のデータを取得済みとし、変更後の全項目を PUT で送信する。
> **•** status が sold になった場合は soldAt を未設定時のみ補完する。
> **•** status が sold 以外へ変わる場合は確認ダイアログ後に soldAt=null とする。
> **•** 代表画像指定がある場合は images 配列の isPrimary を正規化する。
> **•** 商品詳細画面上の関連タスク完了切替では、対象 taskId に対して PATCH /api/tasks/:taskId/completion を呼び出し、完了状態と completedAt のみを更新する。
> **•** 画像追加・差し替え・削除成功後は `GET /api/products/:productId` を再取得し、最新の期限付き URL・sortOrder・代表画像状態を再描画する。

## **6.3 QR 販売済更新**

> **•** QR 読取成功後、/api/qr/lookup で商品特定と更新可否判定を行う。
> **•** canSell=true の場合のみ確認ダイアログを表示する。
> **•** /api/qr/sell 実行時に商品最新状態を再読込し、展示中/在庫中であることを再確認する。
> **•** 更新成功時は status=sold, soldAt=未設定時のみ現在時刻, updatedAt=現在時刻 を保存する。
> **•** すでに sold の場合は更新せず ALREADY_SOLD を返す。

## **6.4 商品論理削除**

> **•** 削除確認後に DELETE /api/products/:productId を実行する。
> **•** API は isDeleted=true, deletedAt, updatedAt を更新する。
> **•** 画像とタスクは保持するが、通常 API では参照不可とする。
> **•** ダッシュボード、検索、QR lookup では対象外とする。

## **6.5 カテゴリ/タグ削除判定**

> **•** 削除前に products を参照し、isDeleted=false の商品の categoryId または tagIds に対象が含まれないことを確認する。
> **•** 参照がある場合は CATEGORY_IN_USE / TAG_IN_USE を返す。
> **•** 削除成功時はマスタから物理削除する。

## **8.2 運用上の注意**

> **•** Storage の画像公開方式は不要な公開 URL を避け、認証前提または期限付き URL を基本とする。URL の既定有効期限は 60 分とし、期限切れ時は商品一覧または商品詳細 API を再取得して最新 URL を受け取る。
> **•** Firestore / Cloud Storage のバックアップ・復旧手順は別途運用設計で定義する。
> **•** 将来の履歴管理や売上管理を見据え、operationLogs と products を疎結合に保つ。
