# 変更ワークフロー・チェックリスト

## 変更前
- [ ] 対象機能の設計書を読む
- [ ] 既存型 / schema / 定数 / util を確認する
- [ ] 既存 API 契約と UI 導線を確認する
- [ ] 影響ファイルを洗い出す

## 実装中
- [ ] 最小差分で進める
- [ ] 無関係なリファクタリングを混ぜない
- [ ] shared に寄せるべきものを見極める
- [ ] API 変更時はフロント / shared / docs も整合させる

## 実装後
- [ ] lint
- [ ] typecheck
- [ ] test
- [ ] build:web
- [ ] 401/403/404/400 の扱いを確認
- [ ] 論理削除 / soldAt / 画像 / URL 期限の影響を確認

## 報告時に書くこと
- 何を変えたか
- なぜ変えたか
- どの仕様に基づくか
- どのファイルを触ったか
- どの確認をしたか
- 未対応事項 / 注意点
